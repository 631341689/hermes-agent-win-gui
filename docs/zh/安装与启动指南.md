# Hermes Agent — 安装与启动指南

本文档面向：**从 GitHub 克隆源码、在本地完成环境搭建并启动** 的开发者。更完整的架构与开发规范见仓库根目录的 [AGENTS.md](../../AGENTS.md)。

---

## 1. 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| **Python** | ≥ 3.11 | 必须。建议使用官方 CPython。 |
| **Node.js** | ≥ 20 | 仅当你需要 **Web 仪表盘前端开发（Vite HMR）** 或 **终端 TUI（`ui-tui/`）开发/构建** 时安装。 |
| **Git** | 任意较新版本 | 用于克隆仓库。 |

**操作系统说明：**

- **Linux / macOS**：与 CI、官方脚本一致，推荐作为主开发环境。
- **Windows**：可在本机用 **PowerShell + venv** 运行 Dashboard、Agent、部分网关能力；内嵌终端 Chat 依赖 **Web 扩展** 与 **PTY**（见下文 `[web]`）。若官方一键安装脚本仅覆盖类 Unix，Windows 请以本文 **pip 可编辑安装** 为准。
- **WSL2**：若在 Windows 上需要与 Linux 完全一致的体验，可在 WSL2 内按 Linux 步骤操作。

## 2. 获取源码

```bash
git clone https://github.com/<你的用户名>/hermes-agent.git
cd hermes-agent
```

若你 fork 了上游仓库，将 URL 换成自己的 fork 地址即可。

---

## 3. Python 虚拟环境

在项目根目录（含 `pyproject.toml` 的目录）执行。

### Linux / macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip setuptools wheel
```

### Windows（PowerShell）

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip setuptools wheel
```

> 若激活脚本被策略禁止，可改用 `cmd` 执行 `.\.venv\Scripts\activate.bat`，或先查阅 Microsoft 文档调整 `ExecutionPolicy`。

## 4. 安装 Hermes（pip 可编辑模式）

依赖组合在 `pyproject.toml` 的 `[project.optional-dependencies]` 中定义。按需选择其一（可后续再补装）。

### 4.1 最小可运行（CLI + Agent 核心）

```bash
pip install -e "."
```

适合：先跑通 `hermes` 对话、不接 Dashboard、不接消息网关。

### 4.2 Web 管理面板 + 内嵌 Chat（推荐日常）

```bash
pip install -e ".[web]"
```

包含 FastAPI、uvicorn；在 **Windows** 上会安装 **pywinpty**，供仪表盘内嵌终端使用。

### 4.3 消息网关（Telegram / Discord / Slack / 飞书等）

```bash
pip install -e ".[messaging]"
```

若单独使用飞书/Lark 适配器，也可：

```bash
pip install -e ".[feishu]"
```

（`feishu` extra 含 `lark-oapi` 等，与 `messaging` 不同组合，按平台文档选择。）

### 4.4 开发 / 测试

```bash
pip install -e ".[dev]"
```

### 4.5 全量可选依赖（体积大、耗时长）

```bash
pip install -e ".[all]"
```

> `[all]` 在部分平台会按标记排除个别不兼容包（例如文档中说明的 Matrix 相关限制），若安装失败请改用语义更小的 extra 组合。

安装完成后，确认入口可用：

```bash
hermes --help
```

## 5. 可选：Node 与前端 / TUI

仅在需要 **Dashboard 前端开发** 或 **TUI 源码级开发** 时执行。


```bash
cd ui-tui
npm install
npm run build
cd ..
```

## 6. 首次配置与密钥

Hermes 用户数据目录由环境变量 **`HERMES_HOME`** 控制，默认等价于用户目录下的 **`.hermes`**（各操作系统路径不同，以实际为准）。

1. **向导（推荐首次）**

   ```bash
   hermes setup
   ```

2. **配置文件**：`HERMES_HOME/config.yaml`（非密钥项）。

3. **密钥**：仅写入 **`HERMES_HOME/.env`**（API Key、Token 等），不要将 `.env` 提交到 Git。

4. **模型与工具**：安装完成后可用：

   ```bash
   hermes model
   hermes tools
   hermes doctor
   ```

### 6.1 `HERMES_HOME/.env` 参考（按需复制、勿提交 Git）

以下变量名与常见用法一致，**按你启用的功能填写**；未使用的可整行删除或留空。密钥只放 `.env`，不要写进 `config.yaml`。

| 变量名 | 用途说明 |
|--------|----------|
| `OPENAI_API_KEY` | OpenAI 兼容推理/嵌入（含知识库向量嵌入、部分 MinerU `title_aided` 等）。 |
| `OPENAI_BASE_URL` | 可选。自定义 OpenAI 兼容网关地址（留空则用官方或上游默认）。 |
| `DEEPSEEK_API_KEY` | 使用 DeepSeek 作为对话或 MinerU `llm_aided_model.provider: deepseek` 时需要。 |
| `DEEPSEEK_BASE_URL` | 可选。DeepSeek API 基地址。 |
| `OPENROUTER_API_KEY` | 使用 OpenRouter 作为供应商时需要。 |
| `NOUS_API_KEY` | 使用 Nous Portal / 托管工具网关等相关能力时（与 `hermes setup` / 登录流程一致）。 |
| `TAVILY_API_KEY` | 当 `config.yaml` 里 `web.backend: tavily`（或自动回退到 Tavily）时，网页搜索工具需要。 |
| `FEISHU_APP_ID` / `FEISHU_APP_SECRET` | 飞书（Lark）机器人应用凭证；另可按开放平台要求配置 `FEISHU_ENCRYPT_KEY`、`FEISHU_VERIFICATION_TOKEN` 等。 |
| `FEISHU_DOMAIN` | 常见为 `feishu`（飞书）或 `lark`（国际版 Lark），与租户一致。 |
| `FEISHU_CONNECTION_MODE` | 例如 `websocket`（长连接模式，按适配器文档）。 |
| `FEISHU_ALLOW_ALL_USERS` | 开发/内测时可 `true` 放宽用户校验；生产环境请按安全策略收紧。 |
| `FEISHU_GROUP_POLICY` | 群策略，例如 `open`（见网关与飞书适配器说明）。 |
| `GATEWAY_ALLOW_ALL_USERS` | 网关侧是否允许未绑定用户（开发调试用，生产慎用）。 |
| `HERMES_GATEWAY_DISABLE_STALE_CODE_CHECK` | 设为 `1` 可跳过网关陈旧代码自检（仅当你明确需要时）。 |
| `HERMES_GIT_BASH_PATH` | **Windows**：若终端/子进程需调用 Git Bash，可设为 `bash.exe` 的绝对路径（如 `C:\Program Files\Git\bin\bash.exe`）。 |
| `#MOCK_PLATFORM_ENABLED=true` | 若你本地有 mock 平台脚本可取消注释；仓库默认不提供该变量，按自建工具使用。 |

更全的可选环境变量表见 **`hermes_cli/config.py`** 中的 **`OPTIONAL_ENV_VARS`**（随版本更新）。

### 6.2 `HERMES_HOME/config.yaml` 片段：`terminal`（工作目录）

消息网关、终端类工具会读取 **`terminal.cwd`**（并桥接到进程环境 **`TERMINAL_CWD`**）。请改成**你本机** Hermes 仓库或希望作为默认工作区的目录。

```yaml
terminal:
  backend: local
  modal_mode: auto
  cwd: D:\path\to\your-hermes-checkout   # 必填：改成你的项目根或工作区路径
  timeout: 180
  env_passthrough: []
  shell_init_files: []
  auto_source_bashrc: true             # Linux/macOS 常见；Windows 上按需要调整
```

### 6.3 `HERMES_HOME/config.yaml` 片段：知识库 MinerU、向量检索与网页搜索

- **MinerU**：`root` 指向包含 **`mineru` 包**的目录（官方仓库常见为 `MinerU-master/MinerU-master`）。本 fork 默认**不**将上游 MinerU 树提交到 Git，需自行 clone 后填写路径，或使用环境变量 **`HERMES_MINERU_ROOT`** 覆盖（见 **`docs/zh/knowledge-api.md`**）。
- **`knowledge.retrieval`**：仅影响**向量**知识库检索；`two_stage: false` 为旧版单阶段行为；`lexical_weight: 0` 表示第二阶段仅按向量分排序（仍可先加宽召回）。
- **`web.backend`**：与 `hermes tools` 中网页搜索供应商一致；需与 **`.env` 中对应 API Key** 同时配置。

```yaml
knowledge:
  mineru:
    enabled: true
    root: "D:/path/to/MinerU-master/MinerU-master"   # 含 mineru 包的目录
    backend: pipeline
    parse_method: auto
    lang: ch
    llm_aided_model:
      provider: openai          # openai → 使用 OPENAI_*；deepseek → DEEPSEEK_*
      default: gpt-4o-mini      # title_aided 等辅助步骤用；可留空以回落到顶层 model
  retrieval:
    two_stage: true             # false = 完全旧逻辑（单阶段合并）
    recall_per_kb: 48
    max_candidates: 96
    lexical_weight: 0.3         # 0 = 仅向量分数（仍可用加宽召回）

web:
  backend: tavily               # firecrawl | tavily | exa | parallel（需各自 API Key）
```

知识库 REST 与枚举的完整说明：**[knowledge-api.md](knowledge-api.md)**。

## 7. 启动方式一览

| 场景 | 命令 | 依赖提示 |
|------|------|----------|
| **经典交互 CLI** | `hermes` 或 `hermes chat` | 核心 `pip install -e "."` |
| **Web 管理面板** | `hermes dashboard --no-open` | 需要 `[web]`；默认监听 **http://127.0.0.1:9119**（以你配置为准） |
| **终端 TUI** | `hermes --tui` | 需构建/安装好 `ui-tui` 对应产物；与 Python `tui_gateway` 子进程配合 |
| **消息 Gateway** | `hermes gateway start` | 按平台安装 `[messaging]` / `[feishu]` 等 |

### 7.1 Web 仪表盘（单进程，最常用）

**Linux / macOS**

```bash
source .venv/bin/activate
hermes dashboard --no-open
```

**Windows**

```powershell
.\.venv\Scripts\Activate.ps1
.\.venv\Scripts\hermes.exe dashboard --no-open
```

浏览器访问控制台输出的地址（一般为 `http://127.0.0.1:9119`）。

### 7.2 前后端分离开发

- **终端 1**：同上启动 `hermes dashboard --no-open`。
- **终端 2**：`cd web && npm run dev`，浏览器打开 Vite 提示的端口（常见为 `http://localhost:5173`），由 Vite 将 `/api` 代理到仪表盘后端。细节见 `web/vite.config.ts`。

### 7.3 消息网关（Telegram / 飞书等）

```bash
hermes gateway start
```

具体机器人在飞书开放平台、Telegram BotFather 等的配置，见官方文档与仓库内 Gateway 相关说明。飞书卡片交互需订阅 **`card.action.trigger`** 等事件，并与可访问的网关地址匹配。

## 9. 发布到 GitHub 的检查清单

将本项目推到你自己的 GitHub 仓库前，建议确认：

1. **许可证**：根目录保留 `LICENSE`（本项目为 MIT），与 `pyproject.toml` 声明一致。
2. **不要提交密钥**：确保 `.gitignore` 已忽略 `.env`、`*.pem`、本地数据库或大型缓存；个人 `config.yaml` 若含隐私也不要提交。
3. **大文件与构建产物**：通常不要提交 `venv/`、`.venv/`、`node_modules/`、`web/dist/` 等；README 中说明用 `npm run build` / `pip install` 生成。
4. **子模块与私有依赖**：若你改过 `pyproject.toml` 中带 git URL 的依赖，fork 用户需能访问对应仓库。
5. **文档链接**：若仓库名或组织变更，替换 README / 本文档中的 clone URL 与徽章链接。
6. **说明安装路径**：在仓库 **README.md** 中加一段指向本文档的链接，例如：「中文安装说明见 [docs/zh/安装与启动指南.md](安装与启动指南.md)」。

推送示例：

```bash
git remote add origin https://github.com/<你的用户名>/hermes-agent.git
git branch -M main
git add .
git commit -m "docs: add Chinese install and run guide"
git push -u origin main
```

## 10. 故障排查

| 现象 | 建议 |
|------|------|
| `hermes` 找不到命令 | 确认已 `activate` venv，且 `pip install -e ".[...]"` 成功。 |
| Dashboard 无法启动 | 安装 `".[web]"`；阅读终端报错是否缺端口或依赖。 |
| Windows 内嵌 Chat / TUI gateway 异常 | 安装 `[web]`；查看 `%USERPROFILE%\.hermes\logs\tui_gateway_crash.log`；控制台编码尽量使用 UTF-8。 |
| 飞书卡片按钮报错 | 确认网关可收 `card.action.trigger`、返回合法 JSON；适配器需与飞书卡片 JSON 2.0 行为一致（见当前代码 `gateway/platforms/feishu.py`）。 |

更系统的开发说明：**[AGENTS.md](../../AGENTS.md)**。
